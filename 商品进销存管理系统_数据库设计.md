# 商品进销存管理系统数据库设计

## 1. 设计说明

本数据库设计面向两周 MVP 版本，使用 MySQL 作为核心业务数据存储。设计重点是支撑以下功能：

- 两角色登录与权限控制。
- 商品管理与商品多图上传。
- 多仓库管理。
- 入库、出库和库存流水。
- 库存查询和简单报表。
- 大模型 Text-to-SQL 查询。
- Redis 防超卖测试结果记录。
- WebSocket 大屏数据统计。

## 2. 命名约定

- 表名使用小写英文和下划线。
- 主键统一使用 `id`。
- 时间字段使用 `created_at`、`updated_at`。
- 状态字段使用 `status`。
- 数量字段使用 `quantity`。
- 金额字段本项目第一版暂不设计。

说明：用户表命名为 `app_user`，避免和 MySQL 系统表或关键字产生混淆。

## 3. 表清单

## 5. 字段设计

### 5.1 用户表 app_user

用途：保存系统登录用户和角色。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGINT | 主键 |
| `username` | VARCHAR(50) | 登录名 |
| `password_hash` | VARCHAR(255) | 密码哈希 |
| `nickname` | VARCHAR(50) | 昵称 |
| `avatar_url` | VARCHAR(255) | 头像地址，可为空 |
| `role` | VARCHAR(30) | 角色 |
| `status` | TINYINT | 状态，1 启用，0 禁用 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

角色取值：

- `ADMIN`：系统管理员。
- `WAREHOUSE_ADMIN`：仓库管理员。

### 5.2 商品表 product

用途：保存商品基础信息。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGINT | 主键 |
| `product_code` | VARCHAR(50) | 商品编号 |
| `product_name` | VARCHAR(100) | 商品名称 |
| `category` | VARCHAR(50) | 商品分类 |
| `unit` | VARCHAR(20) | 单位 |
| `safe_stock` | INT | 安全库存 |
| `status` | TINYINT | 状态，1 启用，0 停用 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

说明：商品图片不直接放在商品表，而是通过 `product_image` 支持多图。

### 5.3 商品图片表 product_image

用途：保存商品多图信息。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGINT | 主键 |
| `product_id` | BIGINT | 商品 ID |
| `image_url` | VARCHAR(255) | 图片访问路径 |
| `is_main` | TINYINT | 是否主图，1 是，0 否 |
| `sort_order` | INT | 排序值 |
| `created_at` | DATETIME | 创建时间 |

业务规则：

- 每个商品最多 5 张图片。
- 每个商品必须有 1 张主图。
- 如果没有手动设置主图，默认第一张为主图。
- 商品列表、库存列表、大屏默认展示主图。

### 5.4 仓库表 warehouse

用途：保存仓库基础信息。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGINT | 主键 |
| `warehouse_code` | VARCHAR(50) | 仓库编号 |
| `warehouse_name` | VARCHAR(100) | 仓库名称 |
| `city` | VARCHAR(50) | 城市 |
| `status` | TINYINT | 状态，1 启用，0 停用 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

### 5.5 当前库存表 inventory

用途：保存每个商品在每个仓库的当前库存。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGINT | 主键 |
| `product_id` | BIGINT | 商品 ID |
| `warehouse_id` | BIGINT | 仓库 ID |
| `quantity` | INT | 当前库存 |
| `updated_at` | DATETIME | 更新时间 |

业务规则：

- 同一商品在同一仓库只能有一条库存记录。
- 入库增加库存。
- 出库扣减库存。
- 库存不能小于 0。

### 5.6 入库记录表 stock_in

用途：记录入库操作。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGINT | 主键 |
| `stock_in_no` | VARCHAR(50) | 入库单号 |
| `product_id` | BIGINT | 商品 ID |
| `warehouse_id` | BIGINT | 仓库 ID |
| `quantity` | INT | 入库数量 |
| `operator_id` | BIGINT | 操作人 ID |
| `remark` | VARCHAR(255) | 备注 |
| `created_at` | DATETIME | 创建时间 |

### 5.7 出库记录表 stock_out

用途：记录出库操作。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGINT | 主键 |
| `stock_out_no` | VARCHAR(50) | 出库单号 |
| `product_id` | BIGINT | 商品 ID |
| `warehouse_id` | BIGINT | 仓库 ID |
| `quantity` | INT | 出库数量 |
| `operator_id` | BIGINT | 操作人 ID |
| `remark` | VARCHAR(255) | 备注 |
| `created_at` | DATETIME | 创建时间 |

### 5.8 库存流水表 inventory_flow

用途：记录每一次库存变化。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGINT | 主键 |
| `product_id` | BIGINT | 商品 ID |
| `warehouse_id` | BIGINT | 仓库 ID |
| `change_type` | VARCHAR(30) | 变化类型 |
| `change_quantity` | INT | 变化数量 |
| `before_quantity` | INT | 变化前库存 |
| `after_quantity` | INT | 变化后库存 |
| `source_no` | VARCHAR(50) | 来源单号 |
| `operator_id` | BIGINT | 操作人 ID |
| `created_at` | DATETIME | 创建时间 |

变化类型：

- `STOCK_IN`：入库。
- `STOCK_OUT`：出库。
- `REDIS_DEDUCT`：Redis 扣减演示。

### 5.9 自然语言查询日志表 nl_query_log

用途：记录 Text-to-SQL 查询过程。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGINT | 主键 |
| `user_id` | BIGINT | 提问用户 ID |
| `question` | VARCHAR(500) | 用户问题 |
| `model_name` | VARCHAR(100) | 大模型名称 |
| `generated_sql` | TEXT | 模型生成 SQL |
| `checked_sql` | TEXT | 安全校验后的 SQL |
| `check_status` | VARCHAR(30) | 校验状态 |
| `result_summary` | TEXT | 中文总结 |
| `created_at` | DATETIME | 创建时间 |

校验状态：

- `PASSED`：通过。
- `REJECTED`：拒绝。
- `FAILED`：模型调用或执行失败。

### 5.10 Redis 扣减流水表 redis_deduct_log

用途：记录 Redis 防超卖并发测试结果。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGINT | 主键 |
| `product_id` | BIGINT | 商品 ID |
| `warehouse_id` | BIGINT | 仓库 ID |
| `request_id` | VARCHAR(100) | 请求 ID |
| `deduct_quantity` | INT | 扣减数量 |
| `status` | VARCHAR(30) | 扣减状态 |
| `fail_reason` | VARCHAR(255) | 失败原因 |
| `created_at` | DATETIME | 创建时间 |

状态：

- `SUCCESS`：扣减成功。
- `FAILED`：扣减失败。

## 6. 建表 SQL

```sql
CREATE DATABASE IF NOT EXISTS inventory_mvp
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;

USE inventory_mvp;

CREATE TABLE app_user (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  nickname VARCHAR(50) NOT NULL,
  avatar_url VARCHAR(255) NULL,
  role VARCHAR(30) NOT NULL,
  status TINYINT NOT NULL DEFAULT 1,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_app_user_username (username),
  KEY idx_app_user_role (role),
  KEY idx_app_user_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE product (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  product_code VARCHAR(50) NOT NULL,
  product_name VARCHAR(100) NOT NULL,
  category VARCHAR(50) NOT NULL,
  unit VARCHAR(20) NOT NULL,
  safe_stock INT NOT NULL DEFAULT 0,
  status TINYINT NOT NULL DEFAULT 1,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_product_code (product_code),
  KEY idx_product_name (product_name),
  KEY idx_product_category (category),
  KEY idx_product_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE product_image (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  product_id BIGINT NOT NULL,
  image_url VARCHAR(255) NOT NULL,
  is_main TINYINT NOT NULL DEFAULT 0,
  sort_order INT NOT NULL DEFAULT 0,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  KEY idx_product_image_product_id (product_id),
  KEY idx_product_image_main (product_id, is_main),
  CONSTRAINT fk_product_image_product
    FOREIGN KEY (product_id) REFERENCES product(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE warehouse (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  warehouse_code VARCHAR(50) NOT NULL,
  warehouse_name VARCHAR(100) NOT NULL,
  city VARCHAR(50) NOT NULL,
  status TINYINT NOT NULL DEFAULT 1,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_warehouse_code (warehouse_code),
  KEY idx_warehouse_city (city),
  KEY idx_warehouse_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE inventory (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  product_id BIGINT NOT NULL,
  warehouse_id BIGINT NOT NULL,
  quantity INT NOT NULL DEFAULT 0,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_inventory_product_warehouse (product_id, warehouse_id),
  KEY idx_inventory_warehouse_id (warehouse_id),
  KEY idx_inventory_quantity (quantity),
  CONSTRAINT fk_inventory_product
    FOREIGN KEY (product_id) REFERENCES product(id),
  CONSTRAINT fk_inventory_warehouse
    FOREIGN KEY (warehouse_id) REFERENCES warehouse(id),
  CONSTRAINT chk_inventory_quantity_non_negative CHECK (quantity >= 0)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE stock_in (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  stock_in_no VARCHAR(50) NOT NULL,
  product_id BIGINT NOT NULL,
  warehouse_id BIGINT NOT NULL,
  quantity INT NOT NULL,
  operator_id BIGINT NOT NULL,
  remark VARCHAR(255) NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_stock_in_no (stock_in_no),
  KEY idx_stock_in_product_id (product_id),
  KEY idx_stock_in_warehouse_id (warehouse_id),
  KEY idx_stock_in_created_at (created_at),
  CONSTRAINT fk_stock_in_product
    FOREIGN KEY (product_id) REFERENCES product(id),
  CONSTRAINT fk_stock_in_warehouse
    FOREIGN KEY (warehouse_id) REFERENCES warehouse(id),
  CONSTRAINT fk_stock_in_operator
    FOREIGN KEY (operator_id) REFERENCES app_user(id),
  CONSTRAINT chk_stock_in_quantity_positive CHECK (quantity > 0)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE stock_out (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  stock_out_no VARCHAR(50) NOT NULL,
  product_id BIGINT NOT NULL,
  warehouse_id BIGINT NOT NULL,
  quantity INT NOT NULL,
  operator_id BIGINT NOT NULL,
  remark VARCHAR(255) NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_stock_out_no (stock_out_no),
  KEY idx_stock_out_product_id (product_id),
  KEY idx_stock_out_warehouse_id (warehouse_id),
  KEY idx_stock_out_created_at (created_at),

  CONSTRAINT fk_stock_out_warehouse
    FOREIGN KEY (warehouse_id) REFERENCES warehouse(id),
  CONSTRAINT fk_stock_out_operator
    FOREIGN KEY (operator_id) REFERENCES app_user(id),
  CONSTRAINT chk_stock_out_quantity_positive CHECK (quantity > 0)

CREATE TABLE inventory_flow (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  product_id BIGINT NOT NULL,
  warehouse_id BIGINT NOT NULL,
  change_type VARCHAR(30) NOT NULL,
  change_quantity INT NOT NULL,
  before_quantity INT NOT NULL,
  after_quantity INT NOT NULL,

  KEY idx_inventory_flow_product_id (product_id),
  KEY idx_inventory_flow_warehouse_id (warehouse_id),

```

## 7. 关键查询示例

### 7.1 查询商品当前库存

```sql
SELECT
  p.product_name,
  w.warehouse_name,
  i.quantity,
  p.safe_stock
FROM inventory i
JOIN product p ON i.product_id = p.id
JOIN warehouse w ON i.warehouse_id = w.id
WHERE p.status = 1
ORDER BY p.product_name, w.warehouse_name
LIMIT 20;
```

### 7.2 查询低库存商品

```sql
SELECT
  p.product_name,
  w.warehouse_name,
  i.quantity,
  p.safe_stock
FROM inventory i
JOIN product p ON i.product_id = p.id
JOIN warehouse w ON i.warehouse_id = w.id
WHERE i.quantity < p.safe_stock
ORDER BY i.quantity ASC
LIMIT 20;
```

### 7.3 查询最近 7 天出库排行

```sql
SELECT
  p.product_name,
  SUM(s.quantity) AS total_out
FROM stock_out s
JOIN product p ON s.product_id = p.id
WHERE s.created_at >= DATE_SUB(CURDATE(), INTERVAL 7 DAY)
GROUP BY p.id, p.product_name
ORDER BY total_out DESC
LIMIT 10;
```

### 7.4 查询各仓库库存总量

```sql
SELECT
  w.warehouse_name,
  SUM(i.quantity) AS total_quantity
FROM inventory i
JOIN warehouse w ON i.warehouse_id = w.id
GROUP BY w.id, w.warehouse_name
ORDER BY total_quantity DESC;
```

### 7.5 查询商品主图

```sql
SELECT
  p.id,
  p.product_name,
  pi.image_url
FROM product p
LEFT JOIN product_image pi
  ON p.id = pi.product_id AND pi.is_main = 1
WHERE p.status = 1
LIMIT 20;
```

## 8. Text-to-SQL 白名单

大模型生成 SQL 时，只允许访问以下表：

- `product`
- `product_image`
- `warehouse`
- `inventory`
- `stock_in`
- `stock_out`
- `inventory_flow`

不允许访问：

- `app_user`
- `nl_query_log`
- `redis_deduct_log`
- 其他非白名单表

原因：

- `app_user` 包含账号和密码哈希，不应暴露给自然语言查询。
- 日志表不是业务分析核心数据。
- Redis 扣减测试日志属于系统演示数据，不作为普通业务查询范围。

## 9. 初始化数据建议

建议初始化：

- 1 个系统管理员。
- 1 个仓库管理员。
- 3 个仓库：北京仓、上海仓、广州仓。
- 8-12 个商品，最好包含咖啡豆、咖啡机、杯具、耗材等。
- 每个商品 1-3 张图片。
- 每个仓库初始化部分库存。
- 构造最近 7 天的出库数据，方便报表和 Text-to-SQL 演示。

## 10. 数据一致性规则

- 入库时必须同时写入 `stock_in`、更新 `inventory`、写入 `inventory_flow`。
- 出库时必须先检查 `inventory.quantity` 是否充足。
- 出库成功后必须同时写入 `stock_out`、更新 `inventory`、写入 `inventory_flow`。
- `inventory.quantity` 不能小于 0。
- `product_image` 每个商品最多 5 条记录。
- 每个商品应有且只有 1 张主图，该规则由业务代码保证。
- Text-to-SQL 只执行安全校验通过的 `SELECT` SQL。
