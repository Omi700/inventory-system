# 商品进销存管理系统接口设计

## 1. 设计说明

本文档基于《商品进销存管理系统_需求分析.md》《商品进销存管理系统_概要设计.md》和《商品进销存管理系统_数据库设计.md》编写，用于指导两周 MVP 版本的前后端联调与后端开发。

系统采用 Java + Python 双服务：

- Java Spring Boot 主服务：面向前端提供业务 API，负责登录权限、商品、图片、仓库、入库、出库、库存、报表、Redis 防超卖、WebSocket、大模型 SQL 安全校验和 MySQL 查询执行。
- Python FastAPI AI 服务：只供 Java 主服务内部调用，负责调用大模型生成 SQL 和中文总结。

前端只调用 Java 主服务，不直接调用 Python AI 服务。

## 2. 通用约定

### 2.1 服务地址

开发环境建议：

- Java 主服务：`http://localhost:8080`
- Python AI 服务：`http://localhost:8001`
- WebSocket：`ws://localhost:8080/ws/dashboard`

### 2.2 认证方式

除登录接口、静态图片访问外，其余 Java 主服务接口都需要登录。

请求头：

```http
Authorization: Bearer <token>
```

### 2.3 角色枚举

```text
ADMIN              系统管理员
WAREHOUSE_ADMIN    仓库管理员
```

### 2.4 通用响应格式

成功：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

失败：

```json
{
  "code": 40001,
  "message": "参数错误",
  "data": null
}
```

### 2.5 分页响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "records": [],
    "total": 100,
    "page": 1,
    "pageSize": 10
  }
}
```

### 2.6 常用错误码

| 错误码 | 说明 |
|---:|---|
| 0 | 成功 |
| 40001 | 参数错误 |
| 40100 | 未登录或 token 无效 |
| 40300 | 权限不足 |
| 40400 | 数据不存在 |
| 40900 | 业务冲突，例如库存不足 |
| 50000 | 服务内部错误 |
| 50010 | Python AI 服务调用失败 |
| 50020 | 大模型生成 SQL 未通过安全校验 |

### 2.7 状态字段约定

用户状态、商品状态、仓库状态：

```text
1 启用
0 禁用或停用
```

## 3. Java 主服务接口

### 3.1 登录与当前用户

#### 3.1.1 登录

- 方法：`POST`
- 路径：`/api/auth/login`
- 权限：无需登录

请求体：

```json
{
  "username": "admin",
  "password": "123456"
}
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "jwt-token",
    "user": {
      "id": 1,
      "username": "admin",
      "nickname": "系统管理员",
```

关键校验：

- 用户名和密码不能为空。

- 路径：`/api/auth/me`

### 3.4 商品图片管理

仅系统管理员可操作图片。

#### 3.4.1 上传商品图片

- 方法：`POST`
- 路径：`/api/products/{id}/images`
- 权限：`ADMIN`
- 请求类型：`multipart/form-data`

表单字段：

| 字段 | 必填 | 说明 |
|---|---|---|
| `files` | 是 | 图片文件数组 |

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": 11,
      "imageUrl": "/uploads/products/p001-main.png",
      "isMain": 1,
      "sortOrder": 1
    }
  ]
}
```

关键校验：

- 只允许 JPG、PNG。
- 单张图片大小建议不超过 2MB。
- 每个商品最多 5 张图片。
- 如果商品原本没有图片，第一张自动设为主图。

#### 3.4.2 设置主图

- 方法：`PUT`
- 路径：`/api/products/{productId}/images/{imageId}/main`
- 权限：`ADMIN`

说明：

- 将指定图片设为主图。
- 同一商品下其他图片自动取消主图。

#### 3.4.3 调整图片排序

- 方法：`PUT`
- 路径：`/api/products/{id}/images/sort`
- 权限：`ADMIN`

请求体：

```json
{
  "items": [
    {
      "imageId": 11,
      "sortOrder": 1
    },
    {
      "imageId": 12,
      "sortOrder": 2
    }
  ]
}
```

#### 3.4.4 删除商品图片

- 方法：`DELETE`
- 路径：`/api/products/{productId}/images/{imageId}`
- 权限：`ADMIN`

关键校验：

- 删除主图后，如果商品仍有其他图片，系统自动选择排序最小的图片为主图。
- 如果商品只剩一张图片，可以删除，但前端应提示商品将没有主图。

### 3.10 Text-to-SQL

前端调用 Java 主服务，Java 主服务调用 Python AI 服务。Python AI 服务不直接连接 MySQL，不执行 SQL。

#### 3.10.1 自然语言业务查询

- 方法：`POST`
- 路径：`/api/nl-query`
- 权限：`ADMIN`、`WAREHOUSE_ADMIN`

请求体：

```json
{
  "question": "最近 7 天出库最多的是哪几款商品？"
}
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "question": "最近 7 天出库最多的是哪几款商品？",
    "generatedSql": "SELECT p.product_name, SUM(s.quantity) AS total_out FROM stock_out s JOIN product p ON s.product_id = p.id WHERE s.created_at >= DATE_SUB(CURDATE(), INTERVAL 7 DAY) GROUP BY p.id, p.product_name ORDER BY total_out DESC LIMIT 10",
    "checkedSql": "SELECT p.product_name, SUM(s.quantity) AS total_out FROM stock_out s JOIN product p ON s.product_id = p.id WHERE s.created_at >= DATE_SUB(CURDATE(), INTERVAL 7 DAY) GROUP BY p.id, p.product_name ORDER BY total_out DESC LIMIT 10",
    "checkStatus": "PASSED",
    "columns": ["product_name", "total_out"],
    "rows": [
      {
        "product_name": "埃塞俄比亚咖啡豆",
        "total_out": 35
      }
    ],
    "summary": "最近 7 天出库最多的是埃塞俄比亚咖啡豆，共出库 35 袋。"
  }
}
```

关键逻辑：

- Java 主服务读取白名单表结构说明。
- Java 主服务调用 Python AI 服务生成 SQL。
- Java 主服务执行 SQL 安全校验。
- 只允许 `SELECT`。
- 禁止多语句。
- 禁止非白名单表。
- 自动限制 `LIMIT 20`。
- Java 主服务执行 SQL 并保存 `nl_query_log`。

## 5. Python AI 服务内部接口

Python AI 服务只允许 Java 主服务调用，前端不直接访问。

### 5.1 生成 SQL

- 方法：`POST`
- 路径：`/ai/text-to-sql`
- 调用方：Java 主服务

请求体：

```json
{
  "question": "最近 7 天出库最多的是哪几款商品？",
  "schema": "可查询表结构说明",
  "rules": [
    "只能生成 SELECT",
    "必须使用白名单表",
    "必须添加 LIMIT 20"
  ]
}
```

响应：

```json
{
  "sql": "SELECT p.product_name, SUM(s.quantity) AS total_out FROM stock_out s JOIN product p ON s.product_id = p.id WHERE s.created_at >= DATE_SUB(CURDATE(), INTERVAL 7 DAY) GROUP BY p.id, p.product_name ORDER BY total_out DESC LIMIT 10",
  "reason": "用户想查询最近 7 天出库最多的商品"
}
```

说明：

- Python AI 服务只负责生成 SQL，不负责安全兜底。
- Java 主服务必须再次校验 SQL。

### 5.2 生成中文总结

- 方法：`POST`
- 路径：`/ai/summarize-result`

```json
      "product_name": "埃塞俄比亚咖啡豆",

  ]

```

响应：

