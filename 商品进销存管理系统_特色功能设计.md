# 商品进销存管理系统特色功能设计

## 项目定位

第一阶段主线项目：**商品进销存管理系统**。

基础功能包括商品管理、入库管理、出库管理、库存查询、供应商管理和基础报表。为了让项目不只是普通的增删改查，加入一个可展示、可解释、后续可升级的特色功能。

## 特色功能：库存健康诊断与智能补货建议

这个功能的目标是让系统像一个“小型库存分析助手”，不仅告诉用户“现在有多少货”，还告诉用户“哪些货可能出问题，以及为什么”。

### 核心效果

系统每天或每次打开时，自动分析商品库存状态，给每个商品打上健康标签：

- 正常：库存和销量比较稳定。
- 低库存：当前库存低于安全库存。
- 即将缺货：按最近销量估算，未来几天可能卖完。
- 滞销：很久没有出库，库存却还比较多。
- 积压：库存远高于近期销售需求。
- 异常波动：某天出库量明显高于平时。

### 推荐实现的第一版规则

先不用复杂机器学习，第一版用可解释规则就能做出特色：

1. 最近 7 天日均销量 = 最近 7 天出库数量 / 7。
2. 预计可售天数 = 当前库存 / 最近 7 天日均销量。
3. 如果当前库存低于安全库存，标记为“低库存”。
4. 如果预计可售天数小于等于 3 天，标记为“即将缺货”。
5. 如果最近 14 天没有出库且当前库存大于 0，标记为“滞销”。
6. 如果当前库存大于最近 30 天销量的 2 倍，标记为“积压”。
7. 如果某天出库量大于最近 7 天日均销量的 3 倍，标记为“异常波动”。

### 补货建议

系统给出建议补货量，而不是只显示风险：

```text
建议补货量 = 目标库存 - 当前库存
目标库存 = 最近 7 天日均销量 × 目标备货天数
```

第一版可以把目标备货天数设为 7 天。后续可以为不同商品设置不同备货周期。

### 页面展示建议

可以做一个“库存健康看板”，包括：

- 商品名称
- 当前库存
- 最近 7 天销量
- 预计可售天数
- 健康标签
- 建议补货量
- 系统解释

示例解释：

```text
可乐当前库存 12 件，最近 7 天日均销量 5 件，预计 2.4 天后售完，建议补货 23 件。
```

## 为什么这个功能特别

普通进销存系统大多只做“记录”和“查询”，这个功能让系统具备一点分析能力：它能从数据中发现库存风险，并给出原因和建议。这样项目汇报时会更像一个完整的信息管理系统，而不是简单的数据库作业。

## 后续可升级方向

- 加入图表：库存趋势、销量趋势、滞销商品排行。
- 加入 NoSQL：用 Redis 缓存库存健康结果，提高看板访问速度。
- 加入预测：用移动平均、指数平滑或简单机器学习预测销量。
- 加入智能体：让用户用自然语言提问，例如“哪些商品这周可能缺货？”系统自动查询并解释。

## 第一阶段实现边界

先完成“规则版库存健康诊断”，不要一开始就上复杂算法。只要能做到数据真实、规则清晰、结果可解释，就已经比普通进销存项目更有亮点。
